# Agent Instructions

## Branching

Do not create, rename, or switch branches unless explicitly instructed.

All work must remain on the current branch.

## Git Remotes and Pushes

Do not add remotes.

Do not push to GitHub or any remote unless explicitly instructed.

All commits are local-only until push is explicitly authorized.

## Scope Control

Follow the approved phase plan exactly.

Do not introduce future-phase capabilities.

Do not create unplanned directories or files.

## Reporting

Every task report must include:

- files created
- files modified
- validation commands run
- git status summary
- confirmation that no forbidden scope was introduced
